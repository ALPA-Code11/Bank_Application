1. Database khud se role_id nahi de sakta! (Backend/Code tayar karega)
Database ek dabba (container) hai, uske paas apna dimaag nahi hai. Woh khud se decide nahi kar sakta ki jo naya user aaya hai woh Manager hai ya Employee. Yeh kaam tumhara Backend Code (jaise FastAPI, Python, Pydantic) karega.

Jab koi user register karega, toh tum alag-alag API endpoints banaoge ya registration form mein ek option rakhoge.

Agar tumne ek endpoint banaya: /register/employee

Toh tumhare Python code ko pehle se pata hai ki is endpoint par jo bhi aayega, use Employee banana hai.

Tumhara Python code Role Table se Employee ki ID nikalega (jo ki 2 hai) aur User Table mein entry daal dega: role_id = 2.
